# dream-multiskill-v2

Architecture-first rebuild of the Dream MultiSkill trading system.

## Goal

- Re-establish top-level architecture before feature migration.
- Keep only validated modules from the legacy repository via whitelist migration.
- Build explicit links between `constraints` and parallel `workflows`.

## Initial Scope

- `constraints/`: constitution, taxonomy, and machine-readable rules.
- `workflows/`: memory, governance, trading-decision, knowledge, evolution.
- `artifacts/`: architecture evidence, memory outputs, verification reports.
- `docs/`: architecture baseline, migration plan, and execution plans.

## Next Steps

1. Freeze architecture baseline and acceptance gates.
2. Migrate Memory L4 core modules by whitelist.
3. Add architecture alignment checks for A0-A9 collaboration.
