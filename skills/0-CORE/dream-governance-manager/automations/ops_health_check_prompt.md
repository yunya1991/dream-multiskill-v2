## 一、自动化概述

### 1.1 目的

建立标准化的运营流程健康检查机制，通过以下步骤实现公司治理：
1. **治理文件维护** - 定期更新治理文档
2. **链路检查** - 验证各SKILL执行链路
3. **SKILL核查** - 确保各SKILL合规运营
4. **健康评估** - 评估运营流程健康度
5. **处罚执行** - 记录违规并执行处罚

### 1.2 执行周期

| 检查类型 | 周期 | 执行人 |
|:---|:---|:---|
| **运营健康检查** | 72小时 | auto-repair / 治理管理部 |
| **SKILL合规抽查** | 每周 | 运营总监 |
| **治理文档维护** | 每月 | COO |

### 3.4 违规检测规则

```
违规检测逻辑：

V001 - 跳步违规
  IF 问题存在 AND FAQ有解 AND 未查FAQ THEN
    记录V001

V002 - FAQ未查阅
  IF 问题类型 in FAQ_SECTIONS AND 未执行faq_check THEN
    记录V002

V003 - FAQ未更新
  IF FAQ无解 AND 问题已解决 AND 超过48h未更新FAQ THEN
    记录V003

V004 - 归档缺失
  IF 问题已解决 AND 未归档 THEN
    记录V004

V005 - 重复违规
  IF 同类问题违规次数 >= 3 THEN
    记录V005
```
