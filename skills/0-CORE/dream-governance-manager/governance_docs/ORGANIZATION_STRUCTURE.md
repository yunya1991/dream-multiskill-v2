## 五、决策权限矩阵

### 5.1 决策层级

| 层级 | 决策类型 | 权限范围 | 审批要求 |
|:---:|:---|:---|:---|
| **L0** | 日常操作 | 自动执行 | 无需审批 |
| **L1** | 战术调整 | A4/A5权限 | 自动执行 |
| **L2** | 战略变更 | 需A6告警 | COO审批 |
| **L3** | 重大决策 | 系统级变更 | 老板审批 |

### 5.2 跨部门协作流程

```
【日常任务分配】
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 任务理解                                            │
│   ├── 识别交易标的 (inst_id)                                │
│   ├── 识别时间框架 (FREQ)                                   │
│   └── 识别特殊要求                                          │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 2: 任务预判                                            │
│   ├── 检查数据可用性 → 找市场情报部                          │
│   ├── 评估计算复杂度 → 找研究部/风控部                      │
│   └── 确认执行通道 → 找执行部                               │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 3: 分配任务                                            │
│   ├── 情报部：数据采集请求                                   │
│   ├── 研究部：信号评分请求                                   │
│   ├── 风控部：仓位计算请求                                   │
│   └── 执行部：下单执行请求                                   │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 4: 监控执行                                            │
│   ├── 检查各团队返回状态                                     │
│   ├── 识别异常/卡点                                         │
│   └── 必要时介入协调                                         │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
     【结果汇总 → 档案室】
```
