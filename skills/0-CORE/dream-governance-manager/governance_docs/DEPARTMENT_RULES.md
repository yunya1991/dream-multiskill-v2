### 2.2 问题升级规程

```
问题升级路径：

Level 1: 本部门处理
  ↓ (无法解决)
Level 2: 查阅FAQ/治理文档
  ↓ (无解)
Level 3: 查阅MEMORY/历史经验
  ↓ (无解)
Level 4: 升级到COO/运营总监
  ↓ (紧急问题)
Level 5: 升级到老板
```

## 三、部门合规要求

### 3.1 通用合规要求（所有部门）

> ⚠️ **所有SKILL必须遵守以下合规要求**

1. **问题处理四步法**（强制）
   - Step 1: 查FAQ
   - Step 2: 查治理文档
   - Step 3: 联网搜索
   - Step 4: 自主分析

2. **归档要求**
   - 问题解决后必须归档
   - FAQ无解需补充FAQ
   - 经验需写入MEMORY.md

3. **报告要求**
   - 执行结果需报告秘书
   - 重要决策需记录
   - 异常情况需升级

### 3.2 各部门特定规程

#### 调研部 (A1)

| 规程 | 说明 |
|:---|:---|
| **情报验证** | 多源交叉验证，避免单一来源 |
| **时效要求** | 24h内产出调研报告 |
| **合规重点** | 避免重复调研同一主题 |

#### 战略部 (A2)

| 规程 | 说明 |
|:---|:---|
| **假设验证** | 每个假设必须有验证方案 |
| **多情景** | 至少输出2个情景 |
| **合规重点** | 避免脱离A1调研 |

#### 战术部 (A4)

| 规程 | 说明 |
|:---|:---|
| **三层索引** | 大师→联网→实践 |
| **仅操作DEMO** | 绝对禁止操作实盘 |
| **合规重点** | 执行前查FAQ |

#### 执行部 (A5)

| 规程 | 说明 |
|:---|:---|
| **账户检查** | 执行前确认账户 |
| **风险门禁** | 执行前通过A7 |
| **合规重点** | 禁止跳步执行 |

#### 监控部 (A6)

| 规程 | 说明 |
|:---|:---|
| **实时监控** | 每小时扫描 |
| **告警分级** | P0/P1/P2分级响应 |
| **合规重点** | 告警必须记录 |

#### 风控部 (A7)

| 规程 | 说明 |
|:---|:---|
| **门禁检查** | 所有执行前必须通过 |
| **否决权** | 风控否决即否决 |
| **合规重点** | 不可绕过门禁 |

#### 复盘部 (A8)

| 规程 | 说明 |
|:---|:---|
| **每日复盘** | 每日14:00执行 |
| **批评闭环** | 批评→整改→验证 |
| **合规重点** | 客观评价 |

#### 做梦部

| 规程 | 说明 |
|:---|:---|
| **定时运行** | 17:00洞察 |
| **大胆假设** | 不受门禁约束 |
| **合规重点** | 洞察必须归档 |

#### 秘书

| 规程 | 说明 |
|:---|:---|
| **报告分发** | 报告分发到正确目录 |
| **任务跟踪** | 跟踪任务完成状态 |
| **合规重点** | 遵循宪法 |
