## 一、总则

### 1.1 治理目标

建立标准化、可审计、可追溯的公司治理体系，确保所有SKILL和Agent在遇到问题时：
1. **优先复用现有知识**（FAQ → 治理文档 → 历史经验）
2. **降低重复试错成本**（减少重复踩坑）
3. **形成组织记忆**（问题解决后必须归档）

### 1.2 适用范围

| 层级 | 范围 | 约束力 |
|:---|:---|:---:|
| **宪法层** | `dream-constitution/SKILL.md` | 🔴 最高约束，所有决策必须遵循 |
| **治理层** | `dream-governance-manager/` | 🔴 强制执行，违规触发处罚 |
| **部门层** | 各SKILL目录 | 🟠 部门规程，必须遵守 |
| **操作层** | 具体执行脚本 | 🟡 最佳实践，鼓励遵循 |


## 三、问题处理标准流程

### 3.1 问题处理四步法（强制）

```
遇到问题时的标准处理流程：

Step 1️⃣ 查FAQ
    ↓ 有解 → 直接执行 ✓
    ↓ 无解 → Step 2

Step 2️⃣ 查治理文档
    ↓ 有解 → 执行 + 补充FAQ ✓
    ↓ 无解 → Step 3

Step 3️⃣ 联网搜索
    ↓ 有解 → 执行 + 归档经验 ✓
    ↓ 无解 → Step 4

Step 4️⃣ 自主分析
    ↓ 有解 → 执行 + 输出报告 + 归档 ✓
    ↓ 无解 → 升级处理 + 标记待研究
```

### 3.2 违规判定标准

| 违规行为 | 判定条件 | 严重度 |
|:---|:---|:---:|
| **跳步违规** | 问题存在但未按Step 1-4处理 | 🔴 严重 |
| **FAQ缺失** | FAQ无解但未更新FAQ | 🟠 中等 |
| **归档缺失** | 问题解决但未归档 | 🟡 轻微 |
| **重复违规** | 同类问题3次未按流程处理 | 🔴 严重 |

### 3.3 处罚矩阵

| 违规次数 | 处罚措施 | 执行人 |
|:---|:---|:---|
| 第1次 | 警告 + 记录 | 系统自动 |
| 第2次 | 记过一次 + 复盘任务 | 运营总监 |
| 第3次 | 绩效扣分20% + 强制培训 | COO |
| 累计5次 | 触发系统级审计 + 全流程复盘 | 老板 |

## 五、运营健康检查

### 5.1 检查周期

| 检查类型 | 周期 | 负责人 | 输出 |
|:---|:---|:---|:---|
| **日常检查** | 每日 | 各SKILL | 自我检查 |
| **运营健康检查** | 72小时 | 运营总监 | 报告→秘书 |
| **合规审计** | 每周 | COO | 审计报告 |
| **系统级审计** | 累计5次违规 | 老板 | 审计决议 |

### 5.2 健康检查维度

```
运营健康检查六大维度：

1️⃣ 流程合规性
   - 是否按四步法处理问题
   - 是否有跳步行为

2️⃣ 文档更新及时性
   - FAQ是否24h内更新
   - 治理文档是否同步

3️⃣ 违规记录统计
   - 违规次数/类型
   - 违规趋势分析

4️⃣ 跨部门协作效率
   - 交接是否规范
   - 信息传递是否完整

5️⃣ 知识复用率
   - FAQ命中率
   - 重复问题统计

6️⃣ 制度执行度
   - 各SKILL执行情况
   - 异常行为识别
```
