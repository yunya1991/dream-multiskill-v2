## 一、宪法相关 FAQ

### Q1: 什么时候需要查阅宪法？

**A**: 以下场景必须查阅宪法（`~/.workbuddy/skills/dream-constitution/SKILL.md`）：

```
必须查阅宪法的场景：
□ 重要决策前（仓位调整、策略变更、系统架构变更）
□ 老板下达指令时（需宪法审查）
□ 发现系统异常，需要判断是否为流程违规
□ 新部门/SKILL 创建时（需符合宪法规范）
□ 任何"不确定是否合规"的场景
```

**依据**: 宪法§0.1 核心文件定位

---

### Q2: 宪法和其他治理文件的关系？

**A**: 宪法是最高指导文件，其他文件必须与宪法一致：

| 文件 | 与宪法关系 |
|------|------------|
| 工作流程与规范 | 必须基于宪法条款制定 |
| FAQ | 解答宪法执行中的常见问题 |

| 公司组织与架构 | 必须符合宪法中的部门职责定义 |

**依据**: 宪法前言

## 二、违规处理 FAQ

### Q3: 什么是"跳步违规"？

**A**: 跳步违规指未按要求顺序处理问题：

```
正确处理顺序（宪法§11.1）：
Step 1: 查 FAQ → 有解 → 执行 ✓
                  ↓ 无解
Step 2: 查治理文档 → 有解 → 执行 + 补充 FAQ ✓
                      ↓ 无解
Step 3: 联网搜索 → 有解 → 执行 + 归档经验 ✓
                  ↓ 无解
Step 4: 自主分析 → 有解 → 执行 + 输出报告 + 归档 ✓
```

**跳步示例**（违规）：
- ❌ 遇到问题直接联网搜索（跳过 Step 1-2）
- ❌ 遇到问题直接自主分析（跳过 Step 1-3）

**处罚**: V001 跳步违规 → 记过一次 + 补齐 FAQ

**依据**: 宪法§11.1 + dream-governance-manager §三

---

### Q4: FAQ 条目写错了或过时了怎么办？

**A**: 
- **写错**: 直接编辑 FAQ 条目，在 `<!-- FAQ_ENTRY_START -->` 区块内修正
- **过时**: 标注 `[已过时]` 前缀 + 说明替代方案，**不可删除**
- **删除**: 禁止删除 FAQ 条目（历史踩坑经验永久保留）

**依据**: 宪法§11.4.3

---

### Q5: 累计违规是怎么计算的？

**A**: 按 SKILL/部门维度累计，不同类型违规分别累计：

| 违规类型 | 累计维度 | 触发条件 |
|----------|----------|----------|
| FAQ 相关（V002/V003） | 全局 | 累计 2 次 → 二级违规 |
| 跳步违规（V001） | SKILL 维度 | 同 SKILL 累计 2 次 → 二级违规 |
| 重复违规（V005） | 问题维度 | 同问题踩 2 次 → 二级违规 |
| 伪造记录（V006） | 全局 | 1 次 → 三级违规 |

**依据**: 宪法§11.2 + dream-governance-manager §二

## 三、邮箱投递 FAQ

### Q6: 为什么我的工作没有被处理？

**A**: 最可能的原因是 **未投递到秘书邮箱**：

```
检查清单：
□ 工作完成后是否写入了秘书邮箱？
  路径: ~/.workbuddy/skills/boss-secretary/reports/
□ 文件命名是否规范？
  格式: {部门标识}_{主题}_{YYYYMMDD_HHMMSS}.md
□ 文件内容是否包含优先级标记？
 标记: [P0]/[P1]/[P2]/[P3]
□ 是否等待了足够时间？
  秘书处理频率: 每 2 小时
```

**依据**: 宪法§12.1

---

### Q7: P0 交易问题为什么没有进入邮箱？

**A**: P0 交易问题 **不走邮箱**，直接进入 A1-A5 专属通道：

```
P0 交易触发流程：
1. A6 情报监控检测到 P0 级告警
2. 直接触发 A1→A2→A3→A4→A5 链路
3. 同时微信推送用户（等待 30 分钟）
4. 无回复 → 自主执行
```

**邮箱路由**：
- P0 交易 → 专属通道（不走邮箱）
- P1/P2/P3 → 秘书邮箱 → 调研部邮箱 → 待修复邮箱

**依据**: 宪法§12.1.4 + §12.2

## 四、任务创建 FAQ

### Q8: 什么时候需要创建自动化任务？

**A**: 以下场景必须通过 `dream-task-creator` SKILL 创建任务：

```
必须创建任务的场景：
□ 需要定时执行的工作（每小时/每天/每周）
□ 需要创建一次性提醒/定时执行
□ 工作流需要通过任务实现（不能口头传达）
□ 临时任务需要自动化执行
```

**禁止**:
- ❌ 通过口头传达要求创建任务
- ❌ 未集成 `dream-task-creator` 就创建任务

**依据**: 宪法§13.1

---

### Q9: 任务创建后如何验证？

**A**: 创建后必须执行以下验证：

```
验证清单：
□ 文件是否存在？
  路径: ~/.workbuddy/automations/<task-name>/automation.toml
□ 内容格式是否正确？
  检查: TOML 格式 + 必需字段（name, status, rrule, cwds, prompt）
□ rrule 是否符合规范？
  参考: dream-task-creator SKILL 中的 RRule 模板库
□ 任务是否能够成功执行？
  测试: 手动触发一次验证
```

**依据**: 宪法§13.3

## 五、数据分析前置 FAQ

### Q10: 为什么我的交易决策被拒绝了？

**A**: 最可能的原因是 **缺少数据分析报告**：

```
交易决策前置条件（宪法§2.3）：
□ 趋势报告 (trend_chart.html) 存在且时效 < 24h
□ 市场调研报告 (daily_market_intel_*.md) 存在且时效 < 24h
□ 顾问调查报告 (survey_*.md) 时效满足要求
□ 主要矛盾已识别并标注
```

**解决方案**:
1. 运行数据分析脚本生成趋势报告
2. 确保报告时效 < 24h（过期需重新生成）
3. 重新提交交易决策

**依据**: 宪法§2.3 + §6.2

---

### Q11: 复盘时为什么要求引用趋势报告？

**A**: 复盘必须用数据检验假设，不能"感觉"：

```
数据驱动复盘规则（宪法§2.4）：
□ 复盘必须引用趋势报告 (trend_chart.html)
□ 必须对比"决策时趋势位置" vs "当前趋势位置"
□ 归因必须基于数据，禁止"感觉"类主观判断
□ 无趋势报告的复盘 = 无效复盘，必须重做
```

**依据**: 宪法§2.4

## 六、工程索引 FAQ

### Q12: 工程索引在哪里？怎么用？

**A**: 工程索引有两个位置：

| 索引类型 | 位置 | 用途 |
|----------|------|------|
| 治理工程索引 | `governance_docs/engineering_index.md` | 治理相关文件/模块索引 |
| 全局工程索引 | `.workbuddy/faq/OKX_FAQ.md` 顶部 | 全系统文件/模块索引 |

**使用方式**:
1. 遇到问题 → 先查工程索引 → 定位相关文件和入口
2. 找到文件 → 直接读取 → 解决问题
3. 新文件/模块 → 同步更新工程索引

**依据**: 宪法§0.1 + §11.4.4
