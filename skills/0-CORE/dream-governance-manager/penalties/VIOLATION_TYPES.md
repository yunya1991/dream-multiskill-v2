## 二、违规类型详解

### 2.1 V001 - 跳步违规

| 属性 | 内容 |
|:---|:---|
| **定义** | 问题存在但未按Step 1-4处理，直接跳到Step 4（自主分析） |
| **判定条件** | 存在匹配的FAQ/治理文档，但未查阅就直接联网/分析 |
| **典型场景** | OKX报错→直接联网搜索→未查FAQ |
| **严重度** | 🔴 严重 (P1) |
| **首次处罚** | 警告 + 记录 |
| **二次处罚** | 记过 + 复盘任务 |
| **三次处罚** | 绩效 -20% |

### 2.2 V002 - FAQ未查阅

| 属性 | 内容 |
|:---|:---|
| **定义** | 问题存在但未查阅FAQ就直接处理 |
| **判定条件** | 问题类型在FAQ中存在，但未执行faq_check步骤 |
| **典型场景** | 技术问题→直接问人→未查技术FAQ |
| **严重度** | 🔴 严重 (P1) |
| **首次处罚** | 警告 |
| **二次处罚** | 记过 |
| **三次处罚** | 绩效 -10% |

### 2.3 V003 - FAQ未更新

| 属性 | 内容 |
|:---|:---|
| **定义** | FAQ中没有答案，但未在48小时内补充 |
| **判定条件** | Step 1查FAQ无解 → 问题解决 → 超过48h未更新FAQ |
| **典型场景** | 解决新问题后忘记更新FAQ |
| **严重度** | 🟠 中等 (P2) |
| **首次处罚** | 通知 |
| **二次处罚** | 警告 |
| **三次处罚** | 记过 |

### 2.4 V004 - 归档缺失

| 属性 | 内容 |
|:---|:---|
| **定义** | 问题解决后未归档到FAQ/治理文档/经验库 |
| **判定条件** | 问题已解决但未执行归档操作 |
| **典型场景** | 解决一个问题后继续下一个，不做记录 |
| **严重度** | 🟡 轻微 (P3) |
| **首次处罚** | 记录 |
| **二次处罚** | 通知 |
| **三次处罚** | 警告 |

### 2.5 V005 - 重复违规

| 属性 | 内容 |
|:---|:---|
| **定义** | 同类问题累计3次跳步处理 |
| **判定条件** | 相同问题类型违规3次 |
| **典型场景** | OKX API问题每次都跳步 |
| **严重度** | 🔴 严重 (P0) |
| **首次处罚** | 绩效 -30% |
| **二次处罚** | 强制培训 |
| **三次处罚** | 降级/解聘 |

### 2.6 V006 - 伪造记录

| 属性 | 内容 |
|:---|:---|
| **定义** | 虚报已查阅FAQ/治理文档 |
| **判定条件** | 声称已查但实际未查 |
| **典型场景** | 日志记录"已查FAQ"但FAQ无相关内容 |
| **严重度** | 🔴 严重 (P0) |
| **首次处罚** | 绩效 -50% + 强制复盘 |
| **二次处罚** | 降级 |
| **三次处罚** | 解聘 |

## 四、违规检测规则

### 4.1 自动检测规则

```python
# 违规检测规则
VIOLATION_RULES = {
    'V001': {
        'name': '跳步违规',
        'check': lambda ctx: (
            ctx.problem_exists and
            ctx.faq_has_answer and
            not ctx.faq_was_checked
        ),
        'severity': 'P1'
    },
    'V002': {
        'name': 'FAQ未查阅',
        'check': lambda ctx: (
            ctx.problem_type in FAQ_SECTIONS and
            not ctx.faq_step_executed
        ),
        'severity': 'P1'
    },
    'V003': {
        'name': 'FAQ未更新',
        'check': lambda ctx: (
            ctx.faq_no_answer and
            ctx.problem_solved and
            ctx.time_since_solution > 48 * 3600 and
            not ctx.faq_updated
        ),
        'severity': 'P2'
    },
    'V004': {
        'name': '归档缺失',
        'check': lambda ctx: (
            ctx.problem_solved and
            not ctx.archived
        ),
        'severity': 'P3'
    },
    'V005': {
        'name': '重复违规',
        'check': lambda ctx: (
            ctx.same_type_violations >= 3
        ),
        'severity': 'P0'
    },
    'V006': {
        'name': '伪造记录',
        'check': lambda ctx: (
            ctx.claimed_checked and
            not ctx.actually_checked
        ),
        'severity': 'P0'
    }
}
```

## 五、特殊豁免

### 5.1 豁免条件

以下情况可申请豁免：

| 豁免类型 | 条件 | 审批人 |
|:---|:---|:---|
| 紧急处理 | P0级别紧急事件，需立即处理 | COO |
| 全新问题 | FAQ/治理文档确实无相关内容 | 部门负责人 |
| 系统故障 | 系统层面导致无法查文档 | 运营总监 |
