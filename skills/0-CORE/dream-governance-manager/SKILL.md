## 二、核心功能

### 2.1 运营流程健康检查（72h）

**触发词**: `运营健康检查`、`治理检查`、`合规检查`

**检查流程**:
```
1️⃣ 读取治理文档
   → GOVERNANCE_CHARTER.md
   → COMPLIANCE_MANUAL.md

2️⃣ 读取处罚记录
   → penalty_records.md

3️⃣ 扫描各SKILL
   → 检查SKILL.md是否包含合规章节
   → 检查近期执行日志

4️⃣ 评估六大维度
   → 流程合规性
   → 文档更新
   → 违规统计
   → 协作效率
   → 知识复用
   → 制度执行

5️⃣ 输出报告
   → 工作区 reports/
   → 秘书目录 ~/.workbuddy/skills/boss-secretary/reports/
```

### 2.2 违规检测

**触发词**: `检测违规`、`违规扫描`、`合规扫描`

**检测规则**:
| 违规代码 | 违规名称 | 严重度 |
|:---|:---|:---:|
| V001 | 跳步违规 | 🔴 |
| V002 | FAQ未查阅 | 🔴 |
| V003 | FAQ未更新 | 🟠 |
| V004 | 归档缺失 | 🟡 |
| V005 | 重复违规 | 🔴 |
| V006 | 伪造记录 | 🔴 |

### 2.3 处罚执行

**触发词**: `执行处罚`、`违规处理`、`记录处罚`

**执行流程**:
```
违规判定 → 查找累计 → 确定等级 → 执行处罚 → 记录 → 通知 → 整改跟踪
```

## 三、合规要求章节模板

> ⭐ **每个SKILL的SKILL.md必须包含以下合规章节**

```markdown
---

## 【合规要求】

### §X 问题处理流程

> ⚠️ **合规约束**: 遇到任何问题必须按以下顺序处理：

```
遇到问题
    ↓
Step 1️⃣ 查FAQ
  → WORKSPACE/.workbuddy/faq/OKX_FAQ.md（OKX相关）
  → WORKSPACE/.workbuddy/faq/技术_FAQ.md（技术相关）
  → WORKSPACE/.workbuddy/faq/运营_FAQ.md（运营相关）
    ↓ 有解 → 执行 ✓
    ↓ 无解 → Step 2

Step 2️⃣ 查治理文档
  → ~/.workbuddy/skills/dream-governance-manager/
    ↓ 有解 → 执行 + 补充FAQ ✓
    ↓ 无解 → Step 3

Step 3️⃣ 联网搜索
  → 使用 tavily/agent-reach 搜索
    ↓ 有解 → 执行 + 归档经验 ✓
    ↓ 无解 → Step 4

Step 4️⃣ 自主分析
    ↓ 有解 → 执行 + 输出报告 + 归档 ✓
    ↓ 无解 → 升级处理
```

### §Y 常见问题索引

| 问题类型 | FAQ位置 | 备注 |
|:---|:---|:---|
| OKX API错误 | `faq/OKX_FAQ.md` | CLI命令/API签名 |
| 账户查询问题 | `faq/OKX_FAQ.md` | 权限/配置文件 |
| 技术实现问题 | `faq/技术_FAQ.md` | 脚本/工具问题 |
| 流程协作问题 | `faq/运营_FAQ.md` | 制度/规范问题 |

### §Z 违规处理

| 违规类型 | 判定条件 | 处罚 |
|:---|:---|:---|
| 跳步违规 | 未查FAQ直接联网/分析 | 记过一次 |
| FAQ缺失 | 问题存在但未查阅 | 警告 |
| 归档缺失 | 问题解决但未归档 | 记录 |

---
```

## 六、使用示例

### 6.1 执行健康检查

```
用户: 运营健康检查
助手:
  → 加载 dream-governance-manager SKILL
  → 读取治理文档
  → 执行六大维度检查
  → 生成报告
  → 分发到秘书目录
```

### 6.2 检测违规

```
用户: 检测dream-tactical-validator的合规性
助手:
  → 读取SKILL.md检查合规章节
  → 检查近期执行日志
  → 判定是否违规
  → 记录处罚（如有）
```

### 6.3 改造SKILL

```
用户: 为dream-tactical-validator添加合规要求
助手:
  → 读取当前SKILL.md
  → 追加【合规要求】章节
  → 更新检查清单
  → 保存文件
```
