## 一、档案分类

### 1.1 内部档案（系统亲身经历）

| 档案类型 | 内容 | 存储位置 |
|:---|:---|:---|
| 成功案例 | 历史成功决策、策略 | MEMORY.md + Episodes |

| 档案类型 | 内容 | 获取方式 |
|:---|:---|:---|
| 行业案例 | 类似市场事件历史 | 联网搜索 |
| 最佳实践 | 外部成功经验 | 联网搜索 |
| 风险事件 | 历史风险事件分析 | 联网搜索 |
| 研究报告 | 外部研究报告摘要 | 联网搜索 |

## 二、核心职能

### 2.1 档案获取

```
外部档案获取流程：

1. 识别需要档案的场景
   → 问题涉及新领域/新情况
   → 内部记忆不足
   → 老板指令审查时

2. 发起联网搜索
   → Tavily/Agent Reach搜索类似事件
   → 查找行业历史数据
   → 参考外部研究报告

3. 档案整理入库
   → 生成档案摘要
   → 标注关联问题类型
   → 存入档案临时存储（供调研报告引用）

4. 档案调用
   → 调研时自动检索相关档案
   → 复盘时对照历史案例
```

### 2.2 档案检索

```
档案检索条件：
□ 遇到新问题/新市场
□ 内部记忆不足
□ 需要外部案例支撑
□ 复盘时对照
□ 老板指令审查时

检索结果：
- 相关档案列表
- 档案摘要
- 可借鉴要点
- 风险提示
```
