## 一、SKILL 概述

### 1.1 核心价值

本 SKILL 蒸馏自：
- **《情报分析心理学》**（Psychology of Intelligence Analysis）- Richards J. Heuer Jr.
- **《情报分析：结构化分析方法》**（Structured Analytic Techniques for Intelligence Analysis）- Heuer & Pherson

### 1.2 核心能力

| 能力模块 | 内容 | 用途 |
|:---|:---|:---|
| **认知偏见检查** | 7大认知偏见 + 检测方法 + 缓解措施 | 分析前自检，避免思维陷阱 |
| **结构化分析工具箱** | 8大类、55种方法，精选10种核心工具 | 提供标准化分析工具 |
| **情报验证流程** | 来源可靠性 × 信息可信度矩阵 | 确保情报质量 |
| **不确定性表达规范** | 概率区间 + 置信度表达 | 规范化输出 |
| **情报分级分类** | P0/P1/P2 + 战略/战术/技术 | 情报管理 |

```python
# A6 分析流程示例
def analyze_intelligence(data):
    # 步骤1: 调用 dream-intelligence-analysis SKILL
    use_skill("dream-intelligence-analysis")

    # 步骤4: 红队分析
    red_team_analysis(best_hypothesis)
    
    # 步骤5: 输出（使用规范化表达）
    return format_output(best_hypothesis, confidence="high", probability="65%-85%")
```

## 三、核心工具使用流程

### 3.1 分析前：启动清单

```
☑️ 问题定义是否清晰？
☑️ 是否已识别关键假设？（调用"关键假设检查"工具）
☑️ 是否已生成多种假设？（调用"结构化头脑风暴"工具）
☑️ 是否考虑了认知偏见？
```

### 3.2 分析中：ACH 方法论

```
步骤1: 提出假设 (ACH1) → 使用"结构化头脑风暴"
   ↓
步骤2: 搜集信息 (ACH2) → 使用"来源可靠性 × 信息可信度矩阵"
   ↓
步骤3: 构建矩阵 (ACH3) → 构建"假设-证据矩阵"
   ↓
步骤4: 评估一致性 (ACH4) → 评估每个证据与各假设的一致性
   ↓
步骤5: 评估诊断性 (ACH5) → 识别最具诊断性的证据
   ↓
步骤6: 选择结论 (ACH7) → 选择最一致的假设
   ↓
步骤7: 沟通不确定性 (ACH8) → 用概率区间表达结论
```

### 3.3 分析后：验证与表达

```
☑️ 是否完成了来源可靠性 × 信息可信度评估？
☑️ 是否用概率区间表达了不确定性？（禁止"可能""或许"等模糊词）
☑️ 是否明确标注了置信度？（高/中/低）
☑️ 是否考虑了高影响/低概率事件？（黑天鹅）
☑️ 是否提供了可操作的建议？
```

## 四、10种核心工具详解

### 🔧 工具1：关键假设检查（必须用）

**用途**：分析前识别并验证所有前提假设

**操作步骤**：
```
1. 列出当前分析所依赖的所有前提假设
2. 对每个假设提问："如果这个假设是错误的，结论会怎样？"
3. 验证每个假设的可靠性（来源？证据？）
4. 标记"高风险的假设"（如果错误会导致严重误判）
```

**输出格式**：
```markdown
## 关键假设检查

| 假设 | 可靠性 | 如果错误的影响 | 验证状态 |
|------|--------|----------------|----------|
| 假设1: BTC 减半后必涨 | 低 | 高 | ❌ 未验证 |
| 假设2: 美联储短期内不加息 | 中 | 中 | ⚠️ 部分验证 |
```

---

### 🔧 工具2：结构化头脑风暴（必须用）

**用途**：生成多种可能的假设，避免过早收敛

**操作步骤**：
```
1. 设定时间限制（如15分钟）
2. 数量优先（尽可能多提出假设）
3. 暂缓评判（不批评任何假设）
4. 组合改进（将多个假设合并）
```

**输出格式**：
```markdown
## 假设生成（头脑风暴）

| 假设ID | 假设内容 | 提出依据 |
|---------|----------|----------|
| H1 | BTC 将突破 $80,000 | 技术面突破 + 宏观环境利好 |
| H2 | BTC 将回调至 $70,000 | 过热回调 + 美联储言论 |
| H3 | BTC 将横盘震荡 | 多空博弈 + 缺乏催化剂 |
| H4 | BTC 将暴跌至 $60,000 | 黑天鹅事件（如交易所黑客） |
```

---

### 🔧 工具3：竞争性假设分析 ACH（必须用）

**用途**：系统性评估多个假设，找出最一致的假设

**操作步骤**：
```
1. 构建"假设-证据矩阵"
2. 评估每个证据与各假设的一致性（✅一致 / ⚠️部分一致 / ❌不一致 / ❓未知）
3. 识别"最具诊断性的证据"（能区分不同假设的证据）
4. 选择"与最多证据一致的假设"
```

**输出格式**：
```markdown
## 竞争性假设分析 (ACH)

| 证据 | H1: 突破$80K | H2: 回调$70K | H3: 横盘 | H4: 暴跌$60K |
|------|--------------|---------------|----------|----------------|
| E1: 技术面突破 | ✅ | ❌ | ⚠️ | ❌ |
| E2: 美联储言论 | ✅ | ⚠️ | ⚠️ | ❌ |
| E3: 链上数据 | ✅ | ⚠️ | ✅ | ❌ |
| E4: 黑天鹅风险 | ⚠️ | ⚠️ | ⚠️ | ✅ |

**结论**: H1（突破$80K）与最多证据一致，但需警惕E4（黑天鹅风险）。
```

---

### 🔧 工具4：红队分析（必须用）

**用途**：站在对手视角挑战自己的分析

**操作步骤**：
```
1. 组建"红队"（可以是A6内置的批判者角色）
2. 红队目标：找出分析中的漏洞、偏见、未考虑的情景
3. 红队方法：
   - 质疑前提假设
   - 提出替代解释
   - 模拟对手决策过程
4. 整合红队反馈，改进分析
```

**输出格式**：
```markdown
## 红队分析

**红队挑战1**: "你说BTC会突破$80K，但忽略了美联储可能突然加息的风险。"

**红队挑战2**: "你认为链上数据支持上涨，但巨鲸可能在假突破后砸盘。"

**红队挑战3**: "你假设市场会理性反应，但算法交易可能导致踩踏。"

**改进措施**:
- 增加"美联储加息"情景分析
- 监控巨鲸地址动向
- 设置止损，防止算法踩踏
```

---

### 🔧 工具5：魔鬼代言人（必须用）

**用途**：指定专人提出反对意见，避免群体思维

**与红队分析的区别**：
- **红队分析**：站在对手视角挑战
- **魔鬼代言人**：站在"故意唱反调"的视角挑战（不一定代表对手）

**操作步骤**：
```
1. 指定A6内置的"魔鬼代言人"角色
2. 魔鬼代言人必须提出至少3个反对理由
3. 分析者必须逐一回应反对理由
4. 如果反对理由成立，必须修正分析
```

---

### 🔧 工具6：指标法（必须用）

**用途**：设定关键指标，持续监控

**操作步骤**：
```
1. 识别"关键指标"（能反映假设是否成立的数据）
2. 设定指标的"阈值"（如：BTC跌破$75,000 → H1假设失效）
3. 持续监控指标
4. 指标触发时，重新评估假设
```

**输出格式**：
```markdown
## 关键指标监控

| 指标 | 当前值 | 阈值 | 含义 |
|------|--------|------|------|
| BTC价格 | $77,359 | <$75,000 | H1（突破）失效 |
| FGI | 72 | >80 | 过热，可能回调 |
| 巨鲸转账 | 无明显异常 | 单笔>5,000 BTC | 可能砸盘 |
```

### 🔧 工具7：情景分析法（推荐）

**用途**：预测未来多种可能情景

**操作步骤**：
```
1. 识别"关键不确定性"（如：美联储是否加息？）
2. 构建2×2矩阵（两个关键不确定性 × 两个可能状态）
3. 生成4种情景
4. 分析每种情景的概率和影响
```

**输出格式**：
```markdown
## 情景分析

| 情景 | 美联储加息？ | 巨鲸砸盘？ | 概率 | BTC价格区间 |
|------|--------------|------------|------|--------------|
| 情景1 | 否 | 否 | 60% | $75K - $85K |
| 情景2 | 是 | 否 | 25% | $65K - $75K |
| 情景3 | 否 | 是 | 10% | $60K - $70K |
| 情景4 | 是 | 是 | 5% | $50K - $60K |
```

---

### 🔧 工具8：决策树法（推荐）

**用途**：分析决策路径和可能结果

**操作步骤**：
```
1. 定义决策点（如：是否买入BTC？）
2. 识别可能的结果（如：买入后上涨/下跌）
3. 评估每个结果的概率和收益/损失
4. 计算期望值，选择最佳决策
```

**输出格式**：
```markdown
## 决策树分析

 decision: 是否买入 BTC？
   ├─ 买入
   │   ├─ 上涨（概率65%）→ 收益 +$5,000
   │   └─ 下跌（概率35%）→ 损失 -$3,000
   │   期望值 = 65%×$5,000 + 35%×(-$3,000) = $2,200
   │
   └─ 不买入
       ├─ 上涨（概率65%）→ 损失机会成本 -$5,000
       └─ 下跌（概率35%）→ 避免损失 +$3,000
       期望值 = 65%×(-$5,000) + 35%×$3,000 = -$2,200

**结论**: 买入的期望值更高（$2,200 > -$2,200），建议买入。
```

### 🔧 工具9：高影响/低概率分析法（必须用）

**用途**：分析"黑天鹅"事件

**操作步骤**：
```
1. 识别"高影响/低概率"事件（如：交易所黑客、监管禁令）
2. 评估事件的潜在影响
3. 制定应对预案（即使概率低，也要有预案）
```

**输出格式**：
```markdown
## 黑天鹅风险分析

| 事件 | 概率 | 影响 | 应对预案 |
|------|------|------|----------|
| 交易所黑客 | 5% | 极高（BTC可能暴跌30%） | 分散存储 + 止损 |
| 监管禁令 | 10% | 高（BTC可能下跌20%） | 关注监管动态 |
| 算法踩踏 | 15% | 中（BTC可能下跌10%） | 设置止损 |
```

### 🔧 工具10：事前分析法（推荐）

**用途**：预测"可能出错的地方"

**操作步骤**：
```
1. 假设"分析结论是错误的"
2. 回溯：什么事件导致了错误？
3. 识别"预警信号"
4. 制定"预防措施"
```

**输出格式**：
```markdown
## 事前分析

**假设**: "BTC将突破$80,000"这个结论是错误的。

**回溯**: 什么导致了错误？
- 原因1: 美联储突然加息
- 原因2: 巨鲸砸盘
- 原因3: 算法踩踏

**预警信号**:
- 美联储官员暗示加息
- 巨鲸地址开始转账
- 订单簿深度突然变薄

**预防措施**:
- 监控美联储言论
- 监控巨鲸地址
- 设置止损
```

### 5.1 A6 监控流程（集成情报分析）

```python
# A6 监控流程伪代码

def monitor_intelligence():
    # 步骤1: 调用 dream-intelligence-analysis SKILL
    use_skill("dream-intelligence-analysis")
    
    # 步骤2: 数据收集
    data = collect_data(sources=["OKX API", "Tavily", "Glassnode"])
    
    # 步骤3: 认知偏见检查
    biases_check = check_cognitive_biases(data)

    hypotheses = brainstorming(data)
    
    # 步骤6: 竞争性假设分析 (ACH)

    # 步骤7: 红队分析
    red_team_feedback = red_team_analysis(best_hypothesis)
    
    # 步骤8: 魔鬼代言人
    devil_advocate_feedback = devil_advocate(best_hypothesis)
    
    # 步骤9: 指标监控
    indicators = monitor_indicators(best_hypothesis)
    
    # 步骤10: 输出（使用规范化表达）
    output = format_output(
        hypothesis=best_hypothesis,

        probability="65%-85%",  # 概率区间
        indicators=indicators,
        red_team_feedback=red_team_feedback
    )
    
    return output
```

## 七、常见问题

### Q1: A6 应该多久调用一次 dream-intelligence-analysis SKILL？

**A**: 
- **每次情报分析前**：必须调用
- **每次监控到异常时**：必须调用
- **定期（如每天）**：可选，用于持续验证假设

---

### Q2: 如果 A6 的分析时间有限，应该优先使用哪些工具？

**A**: 优先使用以下 **3个核心工具**：
1. **关键假设检查**（快速识别风险假设）
2. **竞争性假设分析 ACH**（系统性评估假设）
3. **红队分析**（快速发现漏洞）

---

### Q3: 如何判断"认知偏见检查"是否有效？

**A**: 检查以下迹象：
- ✅ 你是否主动寻找了反驳证据？
- ✅ 你是否考虑了多种假设？
- ✅ 你是否站在对手视角思考了？
- ✅ 你的结论是否用概率区间表达了？

如果以上有2个以上"否"，说明认知偏见检查无效。
