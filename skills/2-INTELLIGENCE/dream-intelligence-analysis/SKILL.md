## 一、核心理论框架（来自《情报分析心理学》）

### 1.1 人类思维的三大局限

| 维度 | 局限性描述 | 对情报分析的影响 |
|:---|:---|:---|
| **广度** | 人脑无法处理太多变量 | 决策时只有少数因素起作用，忽略重要变量 |
| **深度** | 人脑无法处理概率数据 | 难以处理不一致的数据，倾向寻找确定性 |
| **时域** | 人脑会被既往数据干扰 | 对新数据不敏感，锚定在早期信息上 |

**关键洞察**：
> "专业知识本身也会受到人类思维过程中普遍存在的分析陷阱的影响。当专家落入这些陷阱时，对专业知识的信心会让影响更加严重。"

### 1.2 认知偏见检查清单

#### 🔴 高频认知偏见（必须检查）

| 偏见类型 | 表现 | 检测方法 | 缓解措施 |
|:---|:---|:---|:---|
| **确认偏见** | 只关注支持自己观点的信息 | 主动寻找反驳证据 | 竞争性假设分析(ACH) |
| **锚定效应** | 过度依赖第一次接收到的信息 | 检查最初假设是否仍然有效 | 关键假设检查 |
| **镜像思维** | 假设对手会按照自己的逻辑行动 | 站在对手视角思考 | 红队分析 |
| **群体思维** | 为了团队和谐而抑制不同意见 | 指定"魔鬼代言人" | 结构化辩论 |
| **过度自信** | 高估自己的判断准确性 | 使用概率区间而非点估计 | 外部视角校准 |
| **可得性启发** | 依赖容易想起的信息 | 检查信息来源是否全面 | 系统性数据收集 |
| **代表性启发** | 基于表面相似性做判断 | 检查深层因果机制 | 因果分析工具 |

### 1.3 情报分析六步法（ACH方法论）

```
定义问题 → 提出假设(ACH1) → 搜集信息(ACH2) → 评估假设(ACH3/4/5) → 选择最可能的假设(ACH7) → 持续监控新信息(ACH6/8)
```

#### 详细步骤

| 步骤 | 名称 | 操作 | 输出 |
|:---|:---|:---|:---|
| **ACH1** | 提出假设 | 列出所有可能的解释 | 假设清单 |
| **ACH2** | 搜集信息 | 收集与假设相关的证据 | 证据矩阵 |
| **ACH3** | 构建矩阵 | 将假设和证据放入矩阵 | 假设-证据矩阵 |
| **ACH4** | 评估一致性 | 评估每个证据与各假设的一致性 | 一致性评分 |
| **ACH5** | 评估诊断性 | 识别最具诊断性的证据 | 诊断性权重 |
| **ACH6** | 监控新信息 | 持续收集可能改变结论的新证据 | 更新后的矩阵 |
| **ACH7** | 选择结论 | 选择最一致的假设 | 分析报告 |
| **ACH8** | 沟通不确定性 | 用概率区间表达结论 | 不确定性陈述 |

## 三、情报验证流程

### 3.1 来源可靠性 × 信息可信度矩阵

| | **信息高度可信** | **信息中度可信** | **信息低度可信** |
|---|:---:|:---:|:---:|
| **来源高度可靠** | ✅ 可直接使用 | ⚠️ 需交叉验证 | ⚠️ 需多方印证 |
| **来源中度可靠** | ⚠️ 需交叉验证 | ⚠️ 需多方印证 | ❌ 谨慎使用 |
| **来源低度可靠** | ⚠️ 需多方印证 | ❌ 谨慎使用 | ❌ 不使用 |

### 3.2 情报验证四步法

```
步骤1: 来源评估（可靠性、专业性、动机）
   ↓
步骤2: 信息交叉验证（至少2个独立来源）
   ↓
步骤3: 逻辑一致性检查（与已知事实是否矛盾）
   ↓
步骤4: 不确定性量化（用概率区间表达）
```

### 3.3 A6 情报验证清单

- [ ] 来源是否可靠？（检查来源历史准确率）
- [ ] 是否有至少2个独立来源印证？
- [ ] 信息是否与已知事实/历史模式矛盾？
- [ ] 信息发布者的动机是什么？（是否存在利益倾向）
- [ ] 是否考虑了 deception 的可能性？
- [ ] 信息的不确定性是否明确表达？

## 四、不确定性表达规范

### 4.1 禁止使用的模糊表述

| ❌ 禁止使用 | 原因 |
|:---|:---|
| "可能" | 概率范围太宽（20%-80%都算"可能"） |
| "或许" | 同上 |
| "有可能" | 同上 |
| "不太可能" | 模糊 |
| "大概率" | 不精确 |

### 4.2 推荐使用的精确表述

| 概率区间 | 推荐表述 | 示例 |
|:---:|:---|:---|
| **85%-95%** | "极高概率" | "极高概率（85%-95%）BTC将突破$80,000" |
| **65%-85%** | "高概率" | "高概率（65%-85%）会发生回调" |
| **45%-65%** | "概率相当" | "多空概率相当（45%-65%）" |
| **15%-35%** | "低概率" | "低概率（15%-35%）会暴跌" |
| **5%-15%** | "极低概率" | "极低概率（5%-15%）发生黑天鹅" |

### 4.3 置信度表达

除了概率区间，还应表达 **置信度**（对概率估计的自信程度）：

| 置信度 | 含义 | 使用场景 |
|:---:|:---|:---|
| **高置信度** | 有足够证据支持 | 多源印证、历史模式清晰 |
| **中置信度** | 证据部分支持 | 单一来源、历史模式模糊 |
| **低置信度** | 证据不足 | 信息稀缺、高度不确定性 |

**完整表述示例**：
> "高概率（65%-85%）BTC将突破$80,000，**高置信度**（3个独立来源印证，技术面+基本面共振）"

## 七、实战应用示例

### 7.1 示例1：BTC 突然暴跌 10% 的情报分析

**步骤1: 关键假设检查**
- 假设1：这是正常回调
- 假设2：这是趋势反转
- 假设3：这是假突破（洗盘）

**步骤2: 结构化头脑风暴**
- 可能原因：美联储言论、巨鲸砸盘、交易所黑客、算法踩踏...

**步骤3: 竞争性假设分析(ACH)**
- 构建假设-证据矩阵
- 评估每个假设与证据的一致性
- 选择最一致的假设

**步骤4: 红队分析**
- 站在空头视角：为什么他们会砸盘？
- 站在交易所视角：是否有内部问题？

**步骤5: 决策树**
- IF 假设1为真 → 策略：持有或加仓
- IF 假设2为真 → 策略：止损或做空
- IF 假设3为真 → 策略：等待反转

**输出格式**：
> "高概率（65%-85%）这是正常回调，**中置信度**（需要更多数据确认）。建议：持有观望，设置止损于$75,000。"
