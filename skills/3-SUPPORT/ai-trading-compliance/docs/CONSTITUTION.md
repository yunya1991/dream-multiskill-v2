# Dream-MultiSkill 系统宪法

## §14 重要变更合规审查 (v2.9, 2026-05-06)

### 14.1 适用范围
所有可能影响交易决策、风险控制、系统行为的变更，必须经由合规审查。

### 14.2 三道门机制
1. **硬门禁检查**: 7项不可妥协的硬性约束
2. **交易前门禁**: 数据完整性、战略门禁、评分门禁、执行风险门禁
3. **影子验证**: 在应用变更前进行影子验证

### 14.3 硬门禁清单
- H001: SSoT 引用检查
- H002: 可复现性检查
- H003: P3 门禁产物检查
- H004: 回滚可执行检查
- H005: 密钥不出域检查
- H006: 环境隔离检查
- H007: 外联控制检查
- H008: 风险扩张检查
- H009: Rollback Plan ID 检查

### 14.4 审计轨迹绑定
所有合规审查必须生成审计记录，包含变更包、决策、时间戳。

### 14.5 回滚与止血
所有变更必须包含回滚计划，高风险变更需人工审批。
