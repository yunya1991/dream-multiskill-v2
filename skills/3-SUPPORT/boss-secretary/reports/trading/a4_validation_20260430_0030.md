## 五、OKX CLI计划委托参数问题解决

### 问题
OKX CLI `okx swap place` 和 `okx swap algo place` 不支持独立的计划委托（trigger order）：
- `okx swap place --ordType trigger` → 错误 51000 (参数错误)
- `okx swap algo place --ordType conditional` → 需要 `tpTriggerPx` 或 `slTriggerPx`（用于TP/SL附件，不是独立计划委托）

### 解决方案
**直接使用OKX REST API**:
- 端点: `POST /api/v5/trade/order-algo`
- 参数: `ordType="trigger"` + `triggerPx` + `orderPx`
- 请求头: 需要添加 `x-simulated-trading: 1`（demo账户）

### 脚本
已创建以下脚本到 `~/.workbuddy/skills/dream-tactical-validator/scripts/`:
1. `okx_conditional_order_api.py` - 放置计划委托（trigger order）
2. `okx_query_algo_order.py` - 查询algo订单状态
