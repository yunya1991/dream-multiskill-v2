## 议题2: 杠杆超标自动阻断机制 🟠 P1

### 2.1 根因分析

| 问题 | 原因 | 证据 |
|:---|:---|:---|
| 杠杆5x→10x | OKX全仓模式：显示杠杆=名义值/frozen | frozen=$473, 名义=$4,784 → 10.1x |
| 实际杠杆5.26x | 全仓实际杠杆=名义值/equity | equity=$909, 名义=$4,784 → 5.26x |
| 系统限制2x未执行 | pretrade-gatekeeper检查的是**设置杠杆**非**实际杠杆** | OKX API lever字段≠实际风险 |
| 杠杆变更无审计 | A5执行器没有杠杆变更日志 | 无历史记录 |

### 2.2 杠杆失控链路

```
用户设置杠杆→ OKX API设置lever=10x → 冻结保证金=frozen(仅占名义10%)
                                              ↓
                               OKX显示"10x"（名义/frozen）
                                              ↓
                               pretrade-gatekeeper检查lever字段→通过
                                              ↓
                               实际杠杆=名义/equity=5.26x（但未检查）
```

**核心问题**: 全仓模式下，OKX的"杠杆"是一个**显示值**而非**风险值**。系统检查了错误的指标。

### 2.3 杠杆门禁规则建议

```python
# pretrade-gatekeeper 杠杆检查规则 (新增)
def check_leverage_gate(position, equity):
    """全仓实际杠杆检查"""
    actual_leverage = (position.avgPx * position.sz * 0.01) / equity
    
    if actual_leverage > 5.0:
        return GATE_REJECT, f"实际杠杆{actual_leverage:.1f}x超过5x上限"
    elif actual_leverage > 3.0:
        return GATE_WARN, f"实际杠杆{actual_leverage:.1f}x偏高，建议减仓"
    else:
        return GATE_PASS, f"实际杠杆{actual_leverage:.1f}x合规"
```

### 2.4 OKX API杠杆操作

```bash
# 查询当前杠杆设置
okx swap get-leverage --instId BTC-USDT-SWAP --mgnMode cross

# 设置杠杆(全仓模式)——注意: 只影响可开仓量，不影响现有仓位
okx swap leverage --instId BTC-USDT-SWAP --lever 5 --mgnMode cross
```
