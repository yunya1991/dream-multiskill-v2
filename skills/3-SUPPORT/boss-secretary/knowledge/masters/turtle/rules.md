# 海龟交易法则 (Turtle Trading Rules)
> **大师**: Richard Dennis & William Eckhardt  
> **著作**: "Turtle Trading" (1987)  
> **类型**: 趋势跟踪系统  
> **适用市场**: 期货、外汇、加密货币  
> **状态**: 已蒸馏 v1.0

---

## 1. 核心理念

### 1.1 基本信念

> "交易是可以教授的。成功的交易者不是天生的，而是后天培养的。"

**核心观点**:
- 追随趋势，而不是预测趋势
- 长期坚持一致的交易系统
- 风险控制是第一优先级
- 接受小额亏损是捕捉大趋势的成本

### 1.2 市场哲学

| 维度 | 海龟观点 |
|:---|:---|
| **市场本质** | 价格趋势是由供需失衡驱动的 |
| **预测难度** | 短期价格不可预测，长期趋势可跟随 |
| **盈利来源** | 让利润奔跑，截断亏损 |
| **失败成本** | 多次小额亏损是捕捉大趋势的必要代价 |

## 2. 交易系统

### 2.1 入场系统 (Two Systems)

#### 系统一 (短期突破)
```yaml
name: "系统一 - 短期趋势"
entry_type: "突破20日高低点"
stop_loss: "2N (ATR)"
 breakout_period: 20
 N_period: 20
 risk_per_trade: 2%
```

**入场规则**:
1. 价格突破20日最高价 → 做多
2. 价格跌破20日最低价 → 做空
3. 仅在希望之星方向操作

#### 系统二 (长期突破)
```yaml
name: "系统二 - 长期趋势"
entry_type: "突破55日高低点"
stop_loss: "4N (ATR)"
 breakout_period: 55
 N_period: 20
 risk_per_trade: 2%
```

**入场规则**:
1. 价格突破55日最高价 → 做多
2. 价格跌破55日最低价 → 做空
3. 更少的交易，更大的趋势

### 2.2 离场系统

```yaml
离场规则:
  系统一:
    止盈: "10日最低价反向突破"
    止损: "2N (ATR)"
  系统二:
    止盈: "20日最低价反向突破"
    止损: "4N (ATR)"
```

**关键原则**: 从不止盈，除非触发反向信号

### 2.3 仓位管理

#### 波动率调整 (Volatility-Adjusted Position)

```python
# N = ATR(20)
# 单位头寸 = 账户余额 × 0.01 / N

def calculate_unit_size(account_balance, atr, risk_percent=0.01):
    """
    海龟法则核心公式
    账户余额的1% / 波动幅度(N) = 单位头寸
    """
    dollar_volatility = account_balance * risk_percent
    unit_size = dollar_volatility / atr
    return unit_size
```

#### 仓位限制规则

| 规则 | 限制 |
|:---|:---|
| 单市场最大头寸 | 4 单位 |
| 相关市场最大头寸 | 6 单位 |
| 所有多头总头寸 | 12 单位 |
| 所有空头总头寸 | 12 单位 |
| 单一品种总头寸 | 6 单位 |

**简化版 (加密货币)**:
- 单币种最大仓位: 30% (2 单位)
- 总多头仓位: 60%
- 总空头仓位: 30%

## 4. 风险控制

### 4.1 核心风险原则

```
┌─────────────────────────────────────────────────────────────┐
│                    海龟风险金句                               │
├─────────────────────────────────────────────────────────────┤
│  "我会告诉你秘诀：N 的 2% 法则"                               │
│  "永远不要在单笔交易中冒超过 2% 的风险"                       │
│  "亏损是交易的一部分，接受它"                                 │
│  "趋势是你的朋友，直到它结束"                                 │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 风险参数表

| 参数 | 原版海龟 | 加密货币适配 |
|:---|:---:|:---:|
| 单笔风险 | 2% | 2% |
| ATR 周期 | 20日 | 20日 |
| 系统一突破 | 20日 | 20日 |
| 系统二突破 | 55日 | 55日 |
| 系统一止损 | 2N | 2N |
| 系统二止损 | 4N | 3N |
| 最大连续亏损 | 接受 | 接受 |

### 4.3 杠杆使用

```yaml
海龟法则杠杆:
  原版: "20-40倍杠杆"
  原因: "N 很小，需要高杠杆放大"
  加密货币适配:
    BTC: "3-5倍杠杆"

仓位计算公式:
  单位数 = 账户余额 × 2% / (入场价 × ATR百分比)
  总仓位 = 单位数 × 当前价格
```

## 5. 心理法则

### 5.1 海龟心理原则

> Richard Dennis 的名言："你是赚钱还是亏钱，95% 取决于你的心理。"

**核心心理法则**:

| 法则 | 说明 |
|:---|:---|
| **一致性** | 无论感觉如何，坚持系统 |
| **接受亏损** | 亏损是捕捉趋势的成本 |
| **不预测** | 跟随系统，不预测市场 |
| **耐心等待** | 等待正确的信号出现 |
| **纪律执行** | 规则就是规则，没有例外 |

### 5.2 常见心理陷阱

| 陷阱 | 海龟对策 |
|:---|:---|
| **过早止盈** | 用系统止盈，不要人为干预 |
| **逆势加仓** | 只有浮盈时才加仓 |
| **跳过信号** | 一致性 > 单笔盈亏 |
| **过度交易** | 只有系统信号才交易 |
| **报复交易** | 亏损后冷静期 |

## 6. 实施检查清单

### 6.1 系统实施检查

```yaml
实施前检查:
  □ 确认 ATR(20) 计算正确
  □ 确认突破信号正确识别
  □ 确认止损逻辑正确
  □ 确认仓位计算正确
  □ 确认品种流动性足够

入场检查:
  □ 是否在希望之星方向
  □ 当前趋势方向
  □ 波动率是否适中
  □ 相关市场状态

持仓检查:
  □ 当前浮盈/浮亏
  □ 是否触及止损
  □ 是否需要加仓
  □ 是否需要减仓

离场检查:
  □ 是否触发止盈
  □ 是否触及止损
  □ 是否趋势反转
```

### 6.2 每日监控指标

| 指标 | 目标值 | 告警阈值 |
|:---|:---:|:---:|
| 当日亏损 | < 2% | > 2% 停止交易 |
| 连续亏损天数 | - | > 3天 复盘 |
| 单日最大回撤 | < 5% | > 5% 暂停 |
| 趋势持续天数 | - | > 30天 注意反转 |
