## 案例2：ETH/USDT 假突破止损 ❌

### 基本信息

```yaml
案例编号: TURTLE-2026-002
交易标的: ETH/USDT
交易系统: 系统二 (55日突破)
入场时间: 2026-04-03 14:00 UTC+8
出场时间: 2026-04-05 10:00 UTC+8
持仓周期: 2天
```

### 入场分析

```
入场决策:

  背景:
    - ETH 在 $1,850-$1,920 盘整 25 天
    - 突破 $1,920 触发系统二信号

  虚假信号识别:
    - 突破后成交量未明显放大
    - 突破高度仅 0.8% (低于 3% 确认标准)
    - RSI 背离 (价格新高，RSI 未新高)

  问题:
    ⚠️ 未等待确认就入场
    ⚠️ 未验证量能
```

### 仓位计算

```yaml
仓位管理:
  账户余额: $50,000
  单笔风险: 2% = $1,000

  ATR(20): $45 (约2.4%)
  止损幅度: 3N = $135 (适配加密货币)

  仓位计算:
    股数 = $1,000 / $135 ≈ 7.4 ETH
    实际入场: 7 ETH

  入场保证金: 7 × $1,925 = $13,475
  实际仓位: 27% (3倍杠杆)
```

### 持仓与止损

```
持仓记录:

  Day 1 (04-03):
    入场: $1,925
    收盘: $1,905 (-1.04%)
    浮亏: -$140 (-0.28%)
    动作: 持有 (未触及止损)

  Day 2 (04-05):
    开盘: $1,880 (-1.31%)
    触发止损: $1,870
    实际卖出: $1,875
```

### 结果与复盘

```yaml
出场:
  触发条件: 3N 止损
  出场价格: $1,875
  持仓: 7 ETH
  实际亏损: -$350

结果评估:
  亏损: -$350 (-0.7%)
  vs 预期 2%: 控制在预期内
  R-Multiple: -0.35R

教训:
  ❌ 入场前未等待确认
  ❌ 未验证量能是否放大
  ❌ 突破幅度不足就入场

改进:
  □ 增加突破确认标准 (量能 + 幅度)
  □ 盘整时间要求更长
  □ 添加 RSI 背离检查
```

### 仓位计算 (双系统)

```yaml
仓位管理:
  账户余额: $50,000
  单笔风险: 2% = $1,000

  ATR(20): $4.2 (约3.0%)

  系统一仓位:
    止损: 2N = $8.4
    股数 = $1,000 / $8.4 ≈ 119 SOL

  系统二仓位 (共振加仓):
    止损: 3N = $12.6
    股数 = $500 / $12.6 ≈ 40 SOL

  总仓位:
    系统一: 100 SOL ($14,200)
    系统二: 40 SOL ($5,680)
    合计: 140 SOL ($19,880)

  实际仓位: 39.8% (使用约2倍杠杆)
```

### 持仓过程

```
持仓记录:

  Day 1-2 (04-08~09):
    买入: $142 (系统一) + $144 (系统二补仓)
    收盘: $148 (+4.2%)
    浮盈: +$840 (+1.68%)

  Day 3-4 (04-10~11):
    回调: $145 (-2.0%)
    浮盈: +$420 (+0.84%)
    动作: 持有

  Day 5-6 (04-12~13):
    上涨: $158 (+9.0%)
    ETF 净流入 $13M
    浮盈: +$2,240 (+4.48%)

  Day 7-8 (04-14~16):
    高位震荡: $155-$162
    触发: 20日低点 $152 止损
```

### 结果与总结

```yaml
出场:
  触发条件: 20日最低价反向突破
  出场价格: $155
  持仓: 140 SOL
  实际盈亏: +$1,820

  盈利: +$1,820 (+3.64%)

  ✅ 共振时适当加仓
```

## 综合教训总结

### 从所有案例学到的

| # | 教训 | 案例来源 |
|:---:|:---|:---|
| 1 | 等待突破确认再入场 | 案例2 |
| 2 | 趋势明确时持有 | 案例1, 3, 5 |
| 3 | 双系统共振信号更强 | 案例3 |
| 4 | 连续亏损时保持纪律 | 案例4 |
| 5 | 让利润奔跑 | 案例5 |
| 6 | 止损是保险，不是损失 | 全部 |
| 7 | 一致性 > 单笔盈亏 | 案例4 |

### 海龟法则执行检查

```yaml
入场前检查:
  □ 盘整时间 ≥ 10 天
  □ 突破幅度 ≥ 3%
  □ 量能放大 ≥ 2 倍
  □ RSI 未超买/超卖
  □ 方向与趋势一致

持仓中检查:
  □ 未触发止损
  □ 未出现反转信号
  □ 浮盈可持续
  □ 止损可上移

出场检查:
  □ 触发止盈条件
  □ 触发止损条件
  □ 趋势反转确认
```
